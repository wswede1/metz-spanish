Supabase sync settings

Teacher / setup only. Keeps a cloud copy of adaptive-learning data that lives in this browser’s IndexedDB (MetzSpanishDB).

Security: Use only the publishable Supabase API key on this page and in student sites. Never paste the secret (service_role) key into any website or student HTML/JS. If a secret key was exposed, rotate it in the Supabase dashboard under Project Settings → API.
Google sign-in is required for cloud sync and for the activity_completions table. Row-level security ties student rows to the signed-in user. Teachers: allowlisted accounts in teacher_accounts can open teacher-dashboard.html (publishable key only) to view class completions after you run supabase-migration-teacher-dashboard.sql and add your user row via SQL. Others: use the Supabase Dashboard or a server-side script with service_role (never in the browser).

1. Supabase dashboard (one-time)

  • Authentication → Providers: enable Google. Add the Client ID and secret from Google Cloud Console (OAuth consent + OAuth 2.0 Web client).
  • Authentication → URL configuration: set Site URL to your main student hub origin (e.g. GitHub Pages root). Under Redirect URLs, add every origin students use, including http://localhost:… for local testing and full URLs to …/student_site/index.html if needed.
  • (Recommended) Restrict sign-in to your school Google Workspace: use a Supabase Auth Hook or validate email / hd (hosted domain) in an Edge Function or database trigger so only @yourdistrict.org accounts succeed.

2. Database schema

New project: run supabase-schema.sql from this folder in the SQL Editor.
Existing project that already ran the old open-RLS script: run supabase-migration-auth-rls.sql once, then verify policies in the dashboard.

3. Browser credentials

Project URL is usually https://<project-ref>.supabase.co. Paste the publishable key from Project Settings → API.
